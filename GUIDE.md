# Transcribe — Руководство по установке и использованию

Локальный инструмент для перевода видео и аудио в текст. Работает без интернета после установки модели. Данные никуда не отправляются.

---

## Возможности

- Транскрибация видео (MP4, MKV, AVI, MOV, WEBM) и аудио (MP3, M4A, WAV, OGG, AAC, FLAC)
- Выбор модели Whisper: tiny → large (по умолчанию medium)
- Выбор языка: русский, английский, авто-определение
- Аудиоплеер — воспроизведение прямо в браузере, клик по сегменту перематывает
- Редактирование транскрипта прямо в интерфейсе
- Экспорт в TXT и SRT
- Отмена транскрибации в любой момент
- Автообновление при каждом запуске
- Максимальный размер файла: **4 GB**

## Mac (M1 / M2 / M3)

### Что нужно заранее

Ничего специально устанавливать не требуется. Всё устанавливается автоматически при первом запуске:
- **uv** — менеджер Python-окружений (установится автоматически)
- **Python 3.11** — uv скачает самостоятельно, если не установлен
- **ffmpeg** — нужен для видеофайлов (лаунчер предложит установить через Homebrew, если не найден)

Нужен только **git** — macOS предложит установить Xcode Command Line Tools, если его нет.

---

### Установка (один раз)

**Шаг 1 — Клонировать репозиторий**

```bash
git clone https://github.com/des-e/transcribe.git ~/transcribe
```

> Если `git` не установлен — macOS предложит установить Xcode Command Line Tools. Согласиться, дождаться, повторить команду.

**Шаг 2 — Создать иконку для Dock**

```bash
bash ~/transcribe/make_app.sh
```

В папке `~/transcribe` появится `Transcribe.app`.

**Шаг 3 — Добавить в Dock**

Finder → папка `~/transcribe` → перетащить `Transcribe.app` в Dock.

---

### Первый запуск

1. Кликнуть на иконку в Dock
2. macOS спросит «Открыть?» → нажать **Открыть** (один раз)
3. Откроется Terminal — начнётся скачивание модели Whisper medium (~800 MB, MLX-формат)
4. Прогресс виден в терминале — дождаться завершения
5. Откроется браузер с приложением

> Скачивание модели происходит **один раз**. Все следующие запуски — секунды.

---

### Ежедневное использование

Кликнуть на `Transcribe.app` в Dock → браузер откроется автоматически.

**Из терминала:**
```bash
bash ~/transcribe/start.sh
```

---

### Обновления

Происходят автоматически при каждом запуске. Ничего делать не нужно.

---

### Устранение проблем (Mac)

**Приложение не открывается (Gatekeeper блокирует)**
```bash
xattr -cr ~/transcribe/Transcribe.app
```

**Пересоздать Transcribe.app (например, после переустановки)**
```bash
bash ~/transcribe/make_app.sh
```

**Порт 8765 занят**
```bash
kill -9 $(lsof -t -i :8765) 2>/dev/null
```

**ffmpeg не установлен — видеофайлы не работают**
```bash
brew install ffmpeg
```
Или скачать готовый бинарник: https://evermeet.cx/ffmpeg/

**Ошибка `git clone`: `xcrun ... incompatible architecture`**

Возникает на M-чипах с устаревшими Xcode Command Line Tools (обычно после переноса с Intel Mac).

```bash
sudo rm -rf /Library/Developer/CommandLineTools
xcode-select --install
```

После установки повторить `git clone`.

## Windows

### Что нужно заранее

Ничего. `start.bat` установит всё автоматически:
- **uv** — скачивается через PowerShell при первом запуске
- **Python 3.11** — uv скачает самостоятельно
- **ffmpeg** — нужен для видеофайлов; при первом запуске лаунчер предложит установить через `winget`

---

### Установка (один раз)

**Вариант А — через Git**

```
git clone https://github.com/des-e/transcribe.git C:\transcribe
```

**Вариант Б — скачать архив (без Git)**

1. Открыть https://github.com/des-e/transcribe
2. Нажать **Code → Download ZIP**
3. Распаковать в `C:\transcribe`

---

### Первый запуск

1. Открыть папку `C:\transcribe`
2. Дважды кликнуть `start.bat`
3. Откроется консоль — автоматически установится uv, затем скачается модель Whisper medium (~1.5 GB)
4. Прогресс виден в консоли — дождаться завершения
5. Откроется браузер с приложением

> Скачивание модели происходит **один раз**. Все следующие запуски — секунды.

---

### Ежедневное использование

Дважды кликнуть `start.bat` в папке проекта.

**Ярлык на рабочем столе:**
Правой кнопкой на `start.bat` → **Отправить → Рабочий стол (создать ярлык)**.

---

### Обновления

Происходят автоматически при каждом запуске (требуется git). Если использовали Вариант Б — обновляйте вручную, скачивая архив заново.

---

### Устранение проблем (Windows)

**Консоль мгновенно закрывается**

Запустить через командную строку, чтобы увидеть ошибку:
```
cd C:\transcribe
start.bat
```

**Порт 8765 занят**
```
netstat -ano | findstr :8765
taskkill /PID <номер> /F
```

**ffmpeg не установлен — видеофайлы не работают**
```
winget install Gyan.FFmpeg
```
После установки закрыть и снова открыть консоль (путь к ffmpeg появится в PATH).

## Linux

### Установка

```bash
git clone https://github.com/des-e/transcribe.git ~/transcribe
bash ~/transcribe/start.sh
```

`start.sh` установит uv автоматически. ffmpeg нужен для видеофайлов:

```bash
sudo apt install ffmpeg        # Ubuntu / Debian
sudo dnf install ffmpeg        # Fedora
sudo pacman -S ffmpeg          # Arch
```

## Поддерживаемые форматы

| Тип | Форматы | Нужен ffmpeg? |
|-----|---------|---------------|
| Видео | MP4, MKV, AVI, MOV, WEBM | Да |
| Аудио | MP3, M4A, WAV, OGG, AAC, FLAC | Нет |

---

## Модели Whisper

| Модель | Скорость (CPU) | Качество | Размер (Windows) | Размер (Mac M-чип) | Рекомендуется |
|--------|----------------|----------|-------------------|--------------------|---------------|
| Tiny | очень быстро | базовое | ~75 MB | ~40 MB | для тестов |
| Base | быстро | стандартное | ~145 MB | ~80 MB | |
| Small | средне | хорошее | ~490 MB | ~250 MB | |
| **Medium** | медленнее | **отличное** | ~1.5 GB | ~800 MB | ✓ по умолчанию |
| Large | долго | максимальное | ~3 GB | ~1.6 GB | для важных записей |

**Apple Silicon (M1/M2/M3):** используется MLX-бэкенд — в 4–8× быстрее CPU. Модель Large соответствует Whisper large-v3.

**Windows / Linux / Intel Mac:** faster-whisper на CPU (int8). Для ускорения через GPU (CUDA) см. раздел ниже.

---

## GPU-ускорение (Windows / Linux, NVIDIA)

По умолчанию транскрибация идёт на CPU. Если есть NVIDIA GPU — faster-whisper поддерживает CUDA, что ускоряет обработку в несколько раз. Функция находится в разработке.
